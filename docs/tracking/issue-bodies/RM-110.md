Campaign 1 epic for separating portable and native codec profiles and completing
the five-codec portable Tier 1 set.

## Outcomes

- `portable-codecs` is the eventual default and has no C/FFI codec dependency.
- `native-codecs` is explicit opt-in and has identical format, limit, fuzz, and
  differential behavior.
- DEFLATE/gzip, zstd, xz/LZMA2, LZ4 frame, and bzip2 decode and encode through
  the shared codec state machine.

## Acceptance

- [ ] CI rejects native codec packages in the portable dependency graph.
- [ ] Both profiles pass one conformance and malformed-input corpus.
- [ ] Streaming memory remains bounded for every encoder and decoder.
- [ ] Async support does not create a separate parser or native-only behavior.
- [ ] Support and security docs state the profile selected by each feature set.
- [ ] Native and portable benchmark results are recorded against reference tools.
