Campaign 1 epic for the high-level safe artifact engine and its extension and
filesystem boundaries.

## Outcomes

- `ArchiveEngine -> ArchiveSession -> inspect / plan / apply` is the preferred
  Rust UX, with `ArchiveInspection`, `EntryDescriptor`, `ExtractionPlan`,
  `ApplyReport`, `CreateOptions`, `Policy`, and `ProviderSet`.
- Plans are non-serializable, bound to their open session and input identity,
  and cannot be applied to a different input.
- Huge archives can use bounded iterators/events without collecting all
  metadata.
- Compile-time `FormatProvider` and `CodecProvider` registration coexists with
  static built-in dispatch.
- `RangeSource` and `AsyncRangeSource` support seek-oriented formats without
  embedding HTTP or cloud SDKs.
- A capability-reporting filesystem contract has a Linux reference adapter and
  reports unsupported restoration instead of silently discarding metadata.

## Acceptance

- [ ] ADRs cover engine/session identity, providers, ranges, and filesystem capabilities.
- [ ] Session/input mismatch and plan replay tests fail safely.
- [ ] Entry and metadata limits apply to both collected and streaming inspection.
- [ ] Sync, futures, and Tokio adapters drive the same parser/codec state machines.
- [ ] Linux tests cover mode/time, links, xattrs/ACL reporting, sparse data, and atomic commit.
- [ ] Public docs position current readers/writers/pipeline as low-level APIs.
