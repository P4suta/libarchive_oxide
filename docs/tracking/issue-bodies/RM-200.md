Campaign 2 epic for an OCI layer engine over tar, tar+gzip, and tar+zstd.

## Outcomes

- Compressed digest and uncompressed diffID are verified in one streaming pass.
- Whiteouts, opaque directories, uid/gid mappings, hardlinks, xattrs, and path
  conflicts become explicit plan operations.
- Layer creation is deterministic across order, timestamps, ownership, PAX
  emission, and padding.
- Byte/range source adapters are available without registry networking,
  authentication, or cloud SDK dependencies.

## Acceptance

- [ ] OCI image-spec layer fixtures and adversarial whiteout/conflict cases pass.
- [ ] Digest mismatch commits no destination changes.
- [ ] 10 GiB layer soak has no payload-proportional memory growth.
- [ ] Deterministic creation reproduces identical bytes and digests.
- [ ] Examples show external HTTP/S3/GCS/Azure adapters without embedding those SDKs.
- [ ] `oxarchive oci` uses the same layer plan and report types.
