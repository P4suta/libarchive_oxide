Campaign 1 epic for a unified `oxarchive` CLI backed by the same high-level
engine as compatibility entry points.

## Outcomes

- `oxarchive inspect`, `plan`, `apply`, `create`, and `verify` expose the engine
  workflow; `oci` is added with RM-200.
- `--json` includes a schema version and is explicitly versioned separately
  from the stable Rust/C APIs.
- `oxtar`, `oxcpio`, `oxcat`, and `oxunzip` remain compatibility routes and do
  not fork archive policy or parsing.

## Acceptance

- [ ] Every command has contract tests for exit code, stdout/stderr, and unsafe paths.
- [ ] Inspection supports bounded/event output for huge archives.
- [ ] Plan/apply mismatch is rejected by the engine, not reimplemented in CLI code.
- [ ] JSON output has fixtures and a visible schema version.
- [ ] Existing CLI behavior has migration notes for intentional breaking changes.
