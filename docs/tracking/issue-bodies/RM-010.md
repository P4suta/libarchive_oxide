Campaign 0 epic for making every public claim and packaged artifact match the
implementation that actually ships.

## Outcomes

- README, crate metadata, security policy, and support matrix distinguish safe
  project code from native transitive codec backends.
- Format support is described at dialect, method, encryption, and metadata
  level; planned formats are never shown as implemented.
- A repository policy test keeps release automation manual-only and
  draft-before-final-publication.
- The exact packaged crates compile in an external consumer workspace, including
  when repository-only tests are absent.

## Acceptance

- [ ] `just release-policy` passes and has negative unit tests for automatic release triggers.
- [ ] `just package-smoke` packages all public crates and compiles their staged sources together.
- [ ] CI requires both checks.
- [ ] The current zstd/xz/LZ4 backend reality is documented accurately.
- [ ] The support matrix names current ZIP, 7z, and ISO limitations.
- [ ] Completion evidence records the validation commands and resulting commit.

## Non-goals

This issue never authorizes a tag, publish, workflow dispatch, Release edit,
asset upload, or final release publication.
