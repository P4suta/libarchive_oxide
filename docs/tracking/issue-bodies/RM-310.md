Campaign 3 epic for a stable C ABI and deliberately limited libarchive
compatibility shim.

## Outcomes

- `libarchive_oxide-c` owns the small unsafe boundary and exposes opaque handles,
  sized versioned structs, explicit allocator/callback contracts, and explicit
  error ownership through generated `archive_oxide.h`.
- A separate compatibility library implements frequently used
  `archive_read_*`, `archive_write_*`, and `archive_entry_*` calls.
- Dangerous `archive_write_disk` semantics are not copied; policy-aware
  extraction is exposed as an explicit oxide API.

## Acceptance

- [ ] The engine crate continues to forbid unsafe code.
- [ ] Header harnesses compile as C11 and C++17 on Linux, Windows, and macOS.
- [ ] x86_64/aarch64 symbol, struct, and constant snapshots are compared in CI.
- [ ] Miri, ABI fuzzing, and C/C++ ownership/error harnesses pass.
- [ ] The supported shim-symbol list and semantic differences are published.
- [ ] C design feedback is reflected in Rust ownership and cancellation APIs.
